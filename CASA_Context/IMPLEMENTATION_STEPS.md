## Step 2: Build `scanner/project_detector.py`

1. Write a `detect(project_path: str) -> dict` function.
2. Walk the root directory (non-recursive, root only) and check for signature files:
   - `requirements.txt` / `pyproject.toml` ‚Üí `"Python"`
   - `package.json` ‚Üí `"JavaScript/TypeScript"`
   - `go.mod` ‚Üí `"Go"`
   - `Cargo.toml` ‚Üí `"Rust"`
   - `pom.xml` / `build.gradle` ‚Üí `"Java"`
   - `*.sln` / `*.csproj` ‚Üí `"C#"`
3. Read `README.md` if it exists. Extract first 500 characters as the project description.
4. Check if `.git/` directory exists to determine `has_git: true/false`.
5. Return a `ProjectProfile` dict.

---

## Step 3: Build `scanner/file_harvester.py`

1. Write a `harvest(project_path: str, profile: dict) -> dict` function.
2. Implement 4 sub-collectors:

   **A. `_get_hot_files()`**
   - Use `os.walk()` to list all files matching language extensions (e.g., `.py`, `.js`, `.go`).
   - Sort by `os.path.getmtime()` (last modified time) descending.
   - Read and return the full content of the top 10 files.
   - Skip `node_modules/`, `.git/`, `__pycache__/`, `venv/`, `dist/`, `build/` directories.

   **B. `_get_git_history()`**
   - Use `subprocess.run(['git', 'log', '--oneline', '-30'], capture_output=True)` in the target project path.
   - Return stdout as a string. Handle gracefully if git is not found.

   **C. `_get_logs()`**
   - Walk the directory looking for `*.log`, `*.err` files.
   - For each file found, read the **last 200 lines** using a tail-read approach.
   - Aggregate all log tails into a single string.

   **D. `_get_configs()`**
   - Look for `config.json`, `settings.py`, `appsettings.json`, `docker-compose.yml`, `.env` (read **keys only, never values**).
   - Return their content/structure.

3. Apply token budget: if total character count > 200,000 chars, trim content in reverse priority order (configs first, then git, then logs, hot files last).

---

## Step 4: Build `scanner/linter_runner.py`

1. Write a `run(project_path: str, profile: dict) -> str` function.
2. Based on `profile["language"]`:
   - **Python:** Try `ruff check {project_path} --output-format json` first. Fall back to `pylint`.
   - **JavaScript/TypeScript:** Try `npx eslint {project_path} --format json`.
   - **Others:** Return `"Linter not configured for this language."`.
3. Parse the JSON output. Extract only the top 20 issues sorted by severity.
4. Format them as a readable string list.
5. Wrap everything in a `try/except` ‚Äî if the linter is not installed or fails, return `"Linter unavailable."` silently.

---

## Step 5: Build `core/context_builder.py`

1. Write an `assemble(profile, harvested, lint_output) -> dict` function.
2. Build the final JSON payload matching the Ingestion Payload Schema from `SCHEMAS.md`.
3. Compute a rough token estimate (characters / 4) and log it to the terminal so the user can see context size.

---

## Step 6: Build `core/nim_client.py`

1. Initialize the `openai.OpenAI` client pointed at the NIM base URL using `NIM_API_KEY` from `.env`.
2. Write a `query(system_prompt: str, user_payload: str) -> str` function.
3. Implement retry logic with exponential backoff:
   - Attempt 1: immediate
   - Attempt 2: wait 5 seconds
   - Attempt 3: wait 15 seconds
   - If all 3 fail: raise with a clear error message.
4. Return the `choices[0].message.content` string from the response.

---

## Step 7: Build `core/reporter.py`

1. Write a `save(report_text: str, project_path: str, model_name: str)` function.
2. Prepend a metadata block to the report:
   ```markdown
   ---
   **Generated by:** Universal Project Improver
   **Date:** YYYY-MM-DD HH:MM UTC
   **Model:** {model_name}
   **Target:** {project_path}
   ---
   ```
3. Save the file as `UPI_Report_YYYY-MM-DD.md` in the target project's root directory.
4. Print the report path to the terminal upon success.

---

## Step 8: Build `main.py` ‚Äî The Orchestrator

1. Set up `argparse` with these arguments:
   - `--project-path` (required): Path to the project to audit.
   - `--dry-run` (optional flag): Print payload and report to terminal without saving anything.
   - `--model` (optional): Override the default NIM model.

2. Implement the full pipeline in order:
   ```python
   profile   = project_detector.detect(args.project_path)
   harvested = file_harvester.harvest(args.project_path, profile)
   lint      = linter_runner.run(args.project_path, profile)
   payload   = context_builder.assemble(profile, harvested, lint)
   report    = nim_client.query(SYSTEM_PROMPT, format_payload(payload))
   reporter.save(report, args.project_path, model_name)
   ```

3. Load `SYSTEM_PROMPT` from the `PROMPTS.md` template (hardcoded string in `main.py`).

## Step 10: Schedule Daily Runs (Windows)

Use Windows Task Scheduler to run UPI every day at 10:00 PM:

```powershell

$trigger = New-ScheduledTaskTrigger -Daily -At "22:00"
Register-ScheduledTask -Action $action -Trigger $trigger -TaskName "UPI Daily Audit"
```
