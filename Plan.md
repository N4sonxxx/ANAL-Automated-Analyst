Below is a complete **build plan + copy/paste prompt** you can give to your AI to build the Dev Agent.

The agent name:

# **RODOK**
## **RObust Developer Orchestrator Kontrol**

Purpose:

> RODOK reads a `Report.md` generated by UPI, understands the recommended improvements, analyzes the local project files, asks NVIDIA NIM for safe code-change plans/patches, applies those patches locally, runs validation, and shows the developer a Git diff for review.

---

# 1. RODOK High-Level Concept

RODOK should work like this:

```txt
UPI generates Report.md
        ↓
RODOK reads Report.md
        ↓
RODOK extracts improvement tasks
        ↓
RODOK inspects affected project files
        ↓
RODOK sends safe context to NVIDIA NIM API
        ↓
NIM returns a fix plan and unified diff patch
        ↓
RODOK validates the patch
        ↓
RODOK applies patch locally
        ↓
RODOK runs tests/linting
        ↓
RODOK shows changed files and Git diff
        ↓
User accepts/commits/rejects
```

Important:

```txt
NVIDIA NIM = AI brain
RODOK = local orchestrator/tool
Project folder = where code is modified
IDE = only shows the file changes afterward
```

---

# 2. Recommended MVP Features

For the first version, RODOK should support:

## Core Features

1. Read `Report.md`
2. Parse improvement tasks from the report
3. Rank tasks by priority
4. Allow user to select which task to fix
5. Inspect relevant files in the project
6. Send context to NVIDIA NIM API
7. Ask NIM to generate:
   - fix plan
   - changed files list
   - unified diff patch
   - validation commands
8. Save patch to disk before applying
9. Apply patch using Git
10. Run validation:
   - `python -m compileall`
   - `ruff check .` if available
   - `pytest` if available
11. Show `git diff`
12. Ask user if they want to commit

# 3. Recommended CLI Commands

RODOK should be a CLI tool.

Example usage:

```bash
python -m rodok inspect Report.md --project .
```

Shows parsed tasks.

```bash
python -m rodok fix Report.md --project . --dry-run
```

Generates a plan and patch but does not apply it.

```bash
python -m rodok fix Report.md --project . --interactive
```

Lets user choose which tasks to fix.

```bash
python -m rodok fix Report.md --project . --task 1
```

Fixes only task 1.

```bash
python -m rodok fix Report.md --project . --critical-only
```

Only fixes critical issues.

```bash
python -m rodok fix Report.md --project . --apply
```

Applies the generated patch.

# 5. Component Responsibilities

## `cli.py`

Handles command-line usage.

Responsibilities:

- Parse arguments
- Display tasks
- Ask for confirmation
- Call orchestrator flow

Recommended library:

```txt
argparse
```

```txt
typer
```

Loads environment variables.

Must support:

```env
AI_PROVIDER=nvidia
NVIDIA_API_KEY=your_key_here
NIM_BASE_URL=https://integrate.api.nvidia.com/v1
NIM_MODEL=your_model_name_here

```

---

## `models.py`

Define data models.

Example:

```python
from dataclasses import dataclass, field
from typing import list

@dataclass
class ReportTask:
    id: int
    priority: str
    title: str
    description: str
    files: list[str] = field(default_factory=list)
    implementation_steps: list[str] = field(default_factory=list)
    expected_impact: str = ""
    effort: str = ""
    raw_section: str = ""

@dataclass
class FixPlan:
    task_id: int
    summary: str
    risk_level: str
    files_to_modify: list[str]
    validation_commands: list[str]
    patch: str
```

---

## `report_parser.py`

Reads `Report.md` and extracts tasks.

It should support headings like:

```md
### 🔴 CRITICAL #1: Monolithic Context Builder
### 🟠 HIGH #2: Redundant Linter Execution
### 🟡 MEDIUM #1: Duplicated Menu Logic
```

It should extract:

- priority
- title
- problem
- affected files
- implementation steps
- expected outcomes
- effort estimate
- raw markdown section

Parser should be tolerant, because report formats may change.

---

## `project_context.py`

Loads relevant source files.

Responsibilities:

- Resolve project root
- Check file exists
- Read affected files from task
- If no files are listed, scan project lightly
- Ignore dangerous/unnecessary files:
  - `.env`
  - `.git`
  - `node_modules`
  - `.venv`
  - `venv`
  - `__pycache__`
  - binary files
  - large files over a safe limit

It should produce context like:

```txt
=== FILE: core/context_builder.py ===
<content>
```

---

## `nim_client.py`

Handles NVIDIA NIM API requests.

Use OpenAI-compatible client:

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://integrate.api.nvidia.com/v1",
    api_key=os.environ["NVIDIA_API_KEY"],
)
```

The model should come from:

```env
NIM_MODEL=...
```

---

## `planner.py`

Given a `ReportTask` and project context, ask the AI to generate a fix plan.

Output should be strict JSON.

The plan should include:

```json
{
  "summary": "...",
  "risk_level": "LOW|MEDIUM|HIGH",
  "files_to_modify": ["..."],
  "validation_commands": ["python -m compileall .", "pytest"],
  "implementation_notes": ["..."]
}
```

---

## `patcher.py`

Responsible for patch generation and patch application.

Important:

RODOK should prefer **unified diff patches**.

Example patch format:

```diff
diff --git a/example.py b/example.py
index 1234567..abcdef0 100644
--- a/example.py
+++ b/example.py
@@ -1,5 +1,7 @@
 def hello():
-    print("hello")
+    print("hello world")
```

RODOK should:

1. Save patch to:

```txt
.rodok/patches/task-1.patch
```

2. Validate patch format
3. Apply patch using:

```bash
git apply --check .rodok/patches/task-1.patch
git apply .rodok/patches/task-1.patch
```

---

## `validator.py`

Runs safe validation commands.

Allowed commands only:

```txt
python -m compileall .
pytest
ruff check .
python -m pytest
```

Do not run arbitrary commands from the AI without checking them against an allowlist.

---

## `git_manager.py`

Handles Git operations.

Responsibilities:

- Check if inside Git repo
- Check working tree status
- Create branch:

```txt
rodok/fix-task-1-context-builder
```

- Show diff
- Commit changes if user approves

Example commit message:

```txt
RODOK: Fix monolithic context builder

Task: CRITICAL #1
Source: Report.md

Generated by RODOK Dev Agent.
```

---

## `safety.py`

This is very important.

RODOK must prevent dangerous AI actions.

Safety rules:

1. Do not modify `.env`
2. Do not modify `.git`
3. Do not delete large folders
4. Do not run unknown shell commands
5. Do not execute AI-generated scripts directly
6. Do not apply patches that touch forbidden files
7. Require confirmation before applying changes
8. Always support `--dry-run`
9. Always show diff after applying
10. Always create a Git branch before modifications

Forbidden paths:

```txt
.env
.git/
.ssh/
*.pem
*.key
id_rsa
id_ed25519
node_modules/
venv/
.venv/
```

Forbidden shell command patterns:

```txt
rm -rf
curl ... | bash
wget ... | bash
sudo
chmod 777
format
del /s
Remove-Item -Recurse
```

# 8. Prompt for Patch Generation Agent

Use this prompt when asking NVIDIA NIM to generate the actual patch:

```txt
You are RODOK, the RObust Developer Orchestrator Kontrol system.

You are a senior software engineer generating a safe unified diff patch for a local project.

You will be given:
1. A Report.md task
2. A fix plan
3. Relevant source files

Your job:
Generate a valid unified diff patch that implements the fix.

Critical rules:
- Output only the unified diff patch.
- Do not output markdown fences.
- Do not explain the patch outside the diff.
- Do not modify forbidden files such as .env, .git, private keys, secrets, or virtual environments.
- Keep changes minimal and reviewable.
- Preserve existing behavior unless the task requires behavior changes.
- If creating a new file, include it in proper git diff format.
- If modifying imports, ensure they are valid.
- Do not include placeholder code.
- Do not include TODOs unless they already existed and remain appropriate.
- Do not generate destructive operations.
- Do not include shell commands.
- The patch must be applicable using git apply.

Report task:
{task}

Fix plan:
{fix_plan}

Relevant source files:
{project_context}

Return only a unified diff patch.
```
